# 毕业设计前端部分技术总结与论文素材

本文档总结了当前计算机专业毕业设计项目中前端部分的核心信息，旨在为撰写毕业论文提供素材和参考。后续仅需根据后端实际实现补充相关内容即可完成系统设计与实现章节的撰写。

## 1. 技术栈选型

本项目前端采用现代化的 Web 技术栈，主要涉及以下技术和框架：

*   **核心框架**：Vue 3 (基于 Composition API，版本 ^3.5.30)
*   **构建工具**：Vite (^7.3.1) - 提供极速的冷启动和热更新(HMR)体验。
*   **UI 组件库**：Element Plus (^2.13.6) - 提供了一套丰富且美观的桌面端组件库，大幅提高开发效率。并使用了 `@element-plus/icons-vue` 图标库。
*   **路由管理**：Vue Router (^5.0.3) - 实现单页面应用(SPA)的路由跳转与权限控制。
*   **状态管理**：Pinia (^3.0.4) - 新一代 Vue 状态管理库，替代 Vuex，提供更扁平化的 API 和更好的 TypeScript 支持。
*   **网络请求**：Axios (^1.14.0) - 基于 Promise 的 HTTP 库，用于与后端 API 接口进行数据交互。
*   **数据可视化**：ECharts (^6.0.0) - 用于 AI 助手等模块的数据图表展示。

## 2. 前端系统架构与目录结构

项目采用了标准的 Vue 3 + Vite 目录结构，主要目录及文件职责划分如下：

```text
├── public/                 # 静态资源目录（不经过打包构建）
├── src/                    # 源代码目录
│   ├── layout/             # 布局组件（分为 AdminLayout 和 UserLayout，区分不同角色视图）
│   ├── router/             # 路由配置（路由表与导航守卫）
│   ├── stores/             # Pinia 状态管理（如 user.js 存储用户登录态）
│   ├── utils/              # 工具类库（如 request.js 封装 Axios）
│   ├── views/              # 页面级视图组件
│   │   ├── admin/          # 管理员端页面
│   │   ├── user/           # 普通用户端页面
│   │   ├── Login.vue       # 登录页面
│   │   └── Register.vue    # 注册页面
│   ├── App.vue             # 根组件
│   └── main.js             # 项目入口文件（挂载 Vue 实例、注册插件）
├── package.json            # 项目依赖及脚本配置
└── vite.config.js          # Vite 构建与开发服务器配置（含代理配置）
```

## 3. 核心功能模块设计

前端系统根据用户角色划分了两套独立的业务视图，通过不同的 Layout 容器承载：

### 3.1 公共模块
*   **身份认证**：用户登录、用户注册。

### 3.2 用户端 (User)
*   **图书借阅**：浏览馆藏图书并进行借阅操作。
*   **图书归还**：查看已借阅的图书记录并进行归还操作。
*   **旧书捐赠**：用户提交旧书捐赠申请，填写相关信息。
*   **知识库**：访问和浏览系统的知识库文档。
*   **AI助手**：与系统 AI 进行对话互动，可能包含基于 ECharts 的数据图表渲染。
*   **个人信息**：查看和修改个人基本信息。

### 3.3 管理员端 (Admin)
*   **图书管理**：对系统内的图书信息进行增删改查。
*   **用户管理**：管理系统内所有注册用户的信息及状态。
*   **借阅管理**：全局管理借阅记录，处理异常或逾期情况。
*   **捐赠管理**：审批用户的旧书捐赠申请，并分配馆藏位置。
*   **知识库管理**：维护系统的知识文档体系，支持文档上传与管理。
*   **AI助手**：管理员视角的 AI 辅助工具。

## 4. 核心技术实现与亮点 (可用于论文"系统实现"章节)

### 4.1 Axios 网络请求封装与拦截器
在 `src/utils/request.js` 中对 Axios 进行了二次封装：
*   **统一前缀与超时**：设置 `baseURL: '/api'` 并配置超时时间。
*   **请求拦截 (Request Interceptor)**：每次发起请求前，自动从 `localStorage` 获取 JWT Token，并添加到请求头 `Authorization: Bearer <token>` 中，实现无感知的鉴权传递。
*   **响应拦截 (Response Interceptor)**：统一拦截后端返回的响应数据。若状态码不为 200，则使用 Element Plus 的 `ElMessage` 统一抛出错误提示。针对 `401` 状态码（未登录或 Token 过期），实现自动清理本地存储并重定向至登录页。

### 4.2 基于 Vue Router 的路由权限控制 (RBAC)
在 `src/router/index.js` 中利用全局前置守卫 `router.beforeEach` 实现了严格的路由鉴权：
*   **页面访问控制**：通过路由元信息 `meta: { requiresAuth: true }` 标记需要登录的页面，未登录用户将被拦截并重定向到 `/login`。
*   **基于角色的访问控制 (RBAC)**：通过 `meta: { role: 'admin' | 'user' }` 区分页面归属。当用户的角色与路由要求的角色不匹配时，自动将其重定向至其所属角色的主页（`/admin` 或 `/user`），防止越权访问。
*   **登录状态保持**：已登录用户尝试访问登录或注册页时，系统会自动将其引导回业务内部。

### 4.3 状态管理与数据持久化
利用 Pinia (`src/stores/user.js`) 对用户的全局状态（如 Token 和 用户信息）进行集中式管理。配合 `localStorage`，确保页面刷新后用户的登录状态不会丢失。

### 4.4 响应式与数据可视化
系统整合了 ECharts 进行数据展示。针对前端常见的响应式布局问题，对 ECharts 容器进行了特殊处理，以适应如 AI 聊天气泡等动态宽度的场景，保证图表自适应且不发生样式错乱（可参考核心记忆中关于 `flex-shrink` 和 `overflow` 的处理技巧）。

## 5. 后续后端开发对接指南

为使前端正常运行，后端（如 Java Spring Boot / Node.js / Python 等）需要提供对应的 API 接口：
*   **接口前缀**：由于前端配置了 `/api` 的请求转发，后端接口基础路径或网关需响应相应请求。
*   **认证机制**：需实现基于 JWT (JSON Web Token) 的认证机制，前端会在 Header 中传递 `Authorization: Bearer <token>`。
*   **统一返回格式**：前端拦截器期望的后端返回数据结构为：
    ```json
    {
      "code": 200,          // 业务状态码 (200 为成功，401 为未授权)
      "message": "成功",    // 提示信息
      "data": {}            // 实际载荷数据
    }
    ```
*   **核心业务接口**：需包括登录注册、图书 CRUD、借阅/归还流转、捐赠审批、知识库文件管理 (`/ai/document/list` 等) 接口。
