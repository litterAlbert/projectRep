# 智慧图书管理系统需求与技术文档

## 1. 项目概述

本项目是在传统图书管理系统基础上，引入大语言模型（LLM）与检索增强生成（RAG）技术，实现文档智能问答、数据可视化分析、个性化推荐等智慧化服务，提升图书馆运营效率与读者体验。

**用户角色**：

登录支持两种用户角色：管理员和读者

输入正确的账号、密码后，系统会自动识别管理员或者用户，并跳转到相应主页。
- **系统管理员**：注册、登录、书籍管理、读者管理、借阅管理、修改个人信息、修改密码、使用AI助手
- **普通用户**：注册、登录、查询图书信息、借阅、预约和归还图书、查看个人借阅记录、使用AI助手

## 3. 功能模块

### 3.1 书籍管理
- **图书信息字段**：ISBN、书名、作者、出版社、出版日期、库存、借出数量、预约数量、馆藏位置。
- **CRUD**：增删改查

### 3.2 读者管理
- **用户信息字段**：用户名、密码、性别、电话、角色（admin,user）。
- **CRUD**：增删改查

### 3.3 借阅、预约和归还图书
- **借书**：库存不为零时， 可借阅，借阅成功后扣减库存，生成借阅记录（借阅日期）
- **还书**：归还时增加库存，更新借阅记录（归还日期）。
- **预约**：库存为0时，读者可预约。

### 3.4 图书检索
- **检索方式**：书名、作者、ISBN模糊匹配，支持组合查询。
- **结果列表**：书名、作者、出版社、库存、借出数量、预约数量、馆藏位置。

### 3.4 AI助手

#### 3.4.1 文档智能问答（RAG）
- **功能**：用户上传文档（PDF/DOCX，≤20MB），系统提取文本、切片、向量化存储。用户可对文档提问，AI结合上下文回答并标注来源。
- **多轮对话**：支持上下文记忆。
- **幻觉控制**：未找到相关信息时明确告知。

#### 3.4.2 数据智能分析与可视化
- **功能**：用户自然语言输入，比如查询2025年3月份借书数量前十的用户，生成折线图，AI 分析意图，生成 SQL 并查询数据库，返回用户指定的图表类型（柱状图/饼图/折线图）及**结构化数据**，前端使用 ECharts 按指定类型渲染图表。
- **支持的查询**：
  - 热门借阅榜（月度/年度Top N）
  - 热门预约榜
  - 读者借阅排行榜
  - 分类借阅占比
  - 个人借阅统计（普通用户）
  --等等
- **技术实现**：构建语义层（数据库视图），使用 LangChain SQL Agent 生成 SQL，正则表达式提取图表类型（柱状图/饼图/折线图）。后端返回 JSON 格式（包含 `chartType` 和 `data`），前端根据 `chartType` 调用 ECharts 对应渲染函数。

#### 3.4.4 会话管理
- **新建会话**：用户可随时创建新的对话会话，开始独立的问答上下文。
- **会话列表**：展示用户历史会话，按最近更新时间倒序排列，支持重命名、删除。
- **会话记录保存**：每个会话自动保存对话历史（用户问题、AI回答、引用来源），支持查看历史会话详情。
- **上下文隔离**：不同会话之间的对话历史互不影响，切换会话时加载对应上下文。
